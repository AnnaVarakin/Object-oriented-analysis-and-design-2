# ☕ Кофе-билдер

**Выполнила:** Варакина Анастасия  
**Группа:** 932320  
**Лабораторная работа №1**

Система для пошагового создания кофейных напитков с использованием паттерна "Строитель" (Builder). Позволяет бариста или клиентам гибко собирать индивидуальные рецепты без дублирования кода и путаницы в ингредиентах.

---

## Проблема

В условиях высокой нагрузки, например в утренний час пик, бариста тратят много времени на запоминание индивидуальных рецептов и последовательности действий для каждого напитка. Ошибки в порядке добавления ингредиентов или забытые компоненты (сироп, корица) приводят к:

- Недовольству клиентов;
- Перерасходу продуктов;
- Снижению скорости обслуживания.

Создание напитков через конструктор с отдельными классами сильно увеличивает время разработки индивидуального или сезонного рецепта.

## Цель проекта

Создать понятный механизм приготовления кофейных напитков, который позволяет бариста или клиентам самостоятельно собирать различные конфигурации без дублирования кода и путаницы в параметрах.

---

## Описание системы

Система позволяет пользователю выбрать тип напитка, после чего пошагово подбираются следующие компоненты:

- **Основа** (тип напитка): эспрессо, капучино, латте, раф.
- **Молоко**: без молока / коровье / банановое / овсяное / миндальное / кокосовое / соевое.
- **Сироп**: карамель / ваниль / фундук / шоколад / кокос / лаванда / мятный.
- **Дозировка сиропа**: 1 / 2 / 3 дозы.
- **Температура**: горячий / теплый / холодный / со льдом.
- **Добавки**: взбитые сливки / мороженое / маршмеллоу.
- **Топпинг**: шоколадный / карамельный / ягодный.
- **Декор**: без декора / корица / какао-порошок.
- **Объем**: 200 мл / 300 мл / 400 мл.
- **Упаковка**: в заведении / с собой / своя кружка.

---

## Архитектура (Паттерн "Строитель")

Для реализации заказа используется паттерн **Builder**:
- **`CoffeeDrinkBuilder`** (абстрактный строитель) — задает интерфейс для пошагового создания напитка.
- **Конкретные строители** (`EspressoBuilder`, `CappuccinoBuilder`, `LatteBuilder`, `RafCoffeeBuilder`) — реализуют логику выбора ингредиентов, их пропорций и совместимости.
  На выходе формируется объект **`CoffeeDrink`**, который можно отобразить в виде чека или карточки заказа с полным описанием рецепта и всех добавок.

### Вывод
Код с паттерном Строитель лучше, потому что он разделяет обязанности: продукт только хранит данные, строители отвечают за сборку каждого напитка, а форма просто вызывает их методы. Это делает код понятным, легко расширяемым (достаточно создать нового строителя) и избавляет от дублирования. В версии без паттерна вся логика сборки размазана по форме с кучей switch, код дублируется, а добавление нового напитка требует изменений в нескольких местах.
