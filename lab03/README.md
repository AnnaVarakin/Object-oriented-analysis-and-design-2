# Лабораторная работа №3
**Паттерн «Посредник» (Mediator)**
---

## Суть

  Программа представляет собой симуляцию муравьиной колонии, где каждый муравей выполняет свою специализированную роль, а вся координация происходит через матку, которая выступает в роли посредника.

### Основные участники:
- **5 типов муравьёв** с уникальным поведением:
  - **Няньки (NannyAnt)** — выращивают муравьев нужных специальностей
  - **Строители (BuilderAnt)** — строят специализированные здания, для расширения колонии
  - **Разведчики (ScoutAnt)** — ищут новые источники еды и строительства, открывают территории
  - **Сборщики (ForagerAnt)** — собирают и приносят еду или палки
  - **Солдаты (SoldierAnt)** — патрулируют территорию и защищают колонию от паука
- **Матка**: управление муравейником

## Реализация паттерна

| Компонент | Реализация | Описание |
|-----------|-------------------|----------|
| **ConcreteMediator** | `QueenAnt` (Матка) | Реализует логику координации: получает сообщения от муравьёв и раздает задания, обновляет ресурсы, принимает решения о создании новых муравьёв |
| **Colleague** (абстрактный класс) | `Ant` | Определяет общий вид всех муравьев |
| **ConcreteColleagues** | `NannyAnt`, `BuilderAnt`, `ScoutAnt`, `ForagerAnt`, `SoldierAnt` | Конкретные типы муравьёв с уникальным поведением |

## Вывод
  Код с паттерном Посредник лучше, потому что все муравьи общаются только через королеву, а не знают о существовании друг друга — это делает систему гибкой и позволяет легко добавлять новых муравьёв без изменения остальных классов. Без паттерна каждый муравей напрямую зависит от остальных, что создаёт , который сложно поддерживать и расширять. Паттерн разделяет ответственность: муравьи выполняют действия, а королева координирует их взаимодействие, соблюдая принцип единой ответственности.
