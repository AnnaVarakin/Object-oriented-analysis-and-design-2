## Описание проблемы

При создании пиксельной графики холст может содержать тысячи отдельных клеток (пикселей). В простой реализации каждая клетка хранит свой цвет, что приводит к перепотреблению потреблению оперативной памяти; замедлению работы программы; снижению производительности при рисовании. 
Например холст размером 100×100 пикселей потребует 10 000 объектов. 

---

## Цель 

Создать редактор пиксельной графики, который позволяет работать, используя минимальное количество объектов в памяти за счет разделения общего состояния между множеством пикселей. 

---

## Описание системы

Система представляет собой графический редактор для создания пиксель-арта: 

| Компонент | Описание |
|-----------|----------|
| **Холст для рисования** | сетка из клеток (пикселей) размером |
| **Палитра цветов** | базовые цвета |
| **Рисование мышкой** | пользователь может рисовать, закрашивая клетки выбранным цветом |
| **Очистка холста** | сброс всех клеток в белый цвет |

Для оптимизации памяти используется паттерн **Приспособленец**, который разделяет объекты на внутреннее и внешнее состояние:

### Внутреннее состояние (разделяемое)
- **Цвет** — хранится в объекте-приспособленце `PixelColor`
- Создается один раз для каждого цвета
- Используется множеством пикселей

### Внешнее состояние (уникальное)
- **Координаты X, Y** — хранятся в объекте `Pixel`
- У каждого пикселя свои координаты

В результате получаем картинку из пикселей, котрую можно использовать для схем вышивки.
