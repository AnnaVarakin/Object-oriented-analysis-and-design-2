## Описание проблемы

Звездное небо может содержать тысячи звезд, которые не все уникальны. В простой реализации каждая ячейка для звезд хранит свой цвет, размер, способность к мерцанию и заполненость что приводит к перепотреблению потреблению оперативной памяти; замедлению работы программы; снижению производительности при рисовании. 
Например небо размером 100×100 пикселей потребует 10 000 отдельных объектов. 

---

## Цель 

Создать редактор звездного неба, который работает, используя минимальное количество объектов в памяти за счет разделения общего состояния между множеством пикселей. 

---

## Описание системы

Система представляет собой полотно, которое имитирует звездное небо: 

| Компонент | Описание |
|-----------|----------|
| **Небо** | сетка из клеток (пикселей)|
| **Палитра цветов** | желтый и белый |
| **Размер** | маленький, средний и большой |
| **Контур** | полая или целая звезда |
| **Мерцание** | возможность звезды сиять |
| **Удаление** | нажать на созданный объект или очистить все поле |
| **Рандом** | генерация случайных созвездий |

Для оптимизации памяти используется паттерн **Приспособленец**, который разделяет объекты на внутреннее и внешнее состояние.
В результате получаем картинку звездного неба.

### Вывод
Реализация с паттерном Приспособленец лучше, потому что она экономит память за счёт разделения общих характеристик звёзд (цвет, размер, контур, мерцание) между всеми объектами, вместо того чтобы хранить их копии для каждой звезды. Это особенно важно при большом количестве звёзд. Код становится более масштабируемым хотя и немного сложнее. Версия без паттерна проще, но при росте числа объектов она приводит к дублированию данных и неэффективному использованию памяти.
